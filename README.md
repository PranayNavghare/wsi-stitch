# New Stitch

Variable-overlap patch scheduling and overlap-aware stitching for whole-slide image pipelines.

This project starts from the design we want for our stitch engine:

- Generate explicit input and output boxes instead of assuming one global overlap.
- Merge predictions by accumulating pixel values and a count/weight map.
- Keep the core NumPy implementation small enough to test and optimize.

## Quick Example

```python
import numpy as np
from stitch_engine import GridSpec, StitchCanvas, plan_grid

spec = GridSpec(
    image_shape=(1024, 768),
    patch_input_shape=(256, 256),
    patch_output_shape=(224, 224),
    stride_shape=(192, 192),
)

tiles = plan_grid(spec)
canvas = StitchCanvas(shape=(768, 1024, 1))

for tile in tiles:
    prediction = np.ones((tile.output_height, tile.output_width, 1), dtype=np.float32)
    canvas.add(prediction, tile.output_box)

result = canvas.finalize()
```

## Variable Overlap

Use `plan_variable_grid` when overlap changes by row, column, or region. It accepts explicit x/y output start positions, so our scheduler can densify patch coverage near tissue edges, seams, fold regions, or model uncertainty.


## Testing With Patch Images Only

If you only have patch images and no slide metadata, you can still test stitching by providing the missing grid geometry yourself. The tool assumes files are already in row-major order after sorting by filename.

```bash
pip install -e ".[image]"
python -m stitch_engine.patch_folder "path/to/patches" "stitched.png" --cols 20 --overlap-x 32 --overlap-y 32
```

Use `--stride-x` and `--stride-y` instead of overlap when you know the exact patch spacing.
For 1024 patches with 50% base overlap:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched.png" --cols 20 --overlap-percent 50
```

For patches generated by `generate_patches.py`, the filenames already contain base coordinates like `idx000000_x000000_y000000.tif`, so use:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched.tif" --from-filenames
```

For a much faster grid preview, use direct mode. It writes one non-overlapping cell per patch instead of averaging all overlaps, so it is faster and needs far less temporary disk:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched.tif" --from-filenames --mode direct
```

For vibration/jitter correction, use alignment mode. Filename `x/y` values are treated as the initial guess, then neighboring overlaps are matched from image content:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched.tif" --from-filenames --mode align --overlap-percent 50 --jitter-percent 10
```

For large jitter tests, keep the fine search reasonable and let the coarse pass cover the larger range:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched_align.tif" --from-filenames --mode align --overlap-percent 50 --jitter-percent 25 --align-coarse-downsample 16 --align-downsample 4 --align-refine-radius 4
```

The default alignment matcher uses phase correlation. To compare against the older brute-force matcher:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched_align.tif" --from-filenames --mode align --align-matcher brute
```

Alignment solves matched offsets globally to reduce row-by-row drift. If the output still looks too close to the base grid, lower the base anchor weight:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched_align.tif" --from-filenames --mode align --overlap-percent 50 --jitter-percent 25 --align-base-weight 0.01
```

To inspect where alignment is connected or skipped, export diagnostics:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched_align.tif" --from-filenames --mode align --align-diagnostics-dir diagnostics
```

The default accurate writer is chunked. It averages overlaps without creating a giant temporary float canvas:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched.tif" --from-filenames --mode average --writer chunked --chunk-size 2048
```

If overlap averaging makes the image look doubled, use the cell writer after alignment. It writes one central cell from each corrected patch instead of averaging the overlap:

```bash
python -m stitch_engine.patch_folder "path/to/patches" "stitched_cell.tif" --from-filenames --mode align --writer cell --cell-padding 32
```

Without coordinates or slide metadata this cannot recover the true placement if patches were sampled irregularly. It is a preview/debug path for grid-like patch folders.
If patches have jittered positions, base overlap alone gives an approximate stitch. Exact stitching needs either saved coordinates or image-based offset estimation between neighboring patches.
